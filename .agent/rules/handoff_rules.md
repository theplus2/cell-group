# 작업 인수인계 규칙

프로젝트 루트에 다음 두 파일을 유지하여 세션 간 컨텍스트를 보존합니다.

## 1. 파일 구조
- **handoff.md**: 현재 상태 및 단기 프로젝트 컨텍스트 (200줄 이하)
- **CHANGELOG.md**: 과거 모든 변경 이력 및 의사결정 배경 (무제한)

## 2. 세션 규칙
### 세션 시작 시 (필수)
- `handoff.md`가 있으면 **반드시 먼저 읽고** 현재 상태를 파악합니다.

### 세션 종료 시 (필수)
- 작업 완료 시 **자동으로** `handoff.md`와 `CHANGELOG.md`를 업데이트합니다.
- 사용자에게 업데이트 여부를 묻지 않고 즉시 수행합니다.

## 3. 문서 구조

### handoff.md
- **현재 상태**: 현재 버전, 빌드 상태, 주요 URL
- **최근 작업**: 최근 1-2개 주요 작업 요약
- **알려진 이슈**: 해결되지 않은 문제점
- **TODO**: 다음에 진행해야 할 작업 목록

### CHANGELOG.md
- 날짜별 모든 변경 이력
- 주요 설계 결정 사유 (Architecture Decision)
- 삭제되거나 변경된 기능의 히스토리
